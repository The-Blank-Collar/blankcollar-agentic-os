# Telegram bot — beginner playbook

The whole loop:

```
You (phone)  →  Telegram  →  ngrok  →  paperclip /api/webhooks/telegram
                                                     ↓
                                        Portkey → Claude → reply
                                                     ↓
You (phone)  ←  Telegram  ←  Telegram Bot API  ←  paperclip
```

End-to-end round-trip: ~3-5 seconds.

## What you need (one-time setup, ~10 min)

- The full local stack running (`make doctor` green) — see `docs/LOCAL_SETUP.md`.
- Portkey configured with an Anthropic virtual key — without this, the
  bot will reply with a friendly "brain unavailable" message instead of
  real LLM output.
- A Telegram account (your phone or `web.telegram.org`).
- A free [ngrok](https://ngrok.com) account — gives Telegram a public
  HTTPS URL pointing at your local paperclip.

## Step 1 — Create a bot with @BotFather

1. Open Telegram.
2. In the search bar, type **`BotFather`** → tap the result with the
   **blue checkmark** (the official one).
3. Tap **START** (or send `/start`).
4. Send `/newbot`.
5. Reply with a display name when asked: e.g. `Blank Collar Bot`.
6. Reply with a username (must end in `bot`): e.g. `blankcollar_test_bot`.
   If taken, try variations.
7. **Copy the token** BotFather gives you. Looks like:
   ```
   8123456789:AAH-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
   ```
   Treat it like a password.

## Step 2 — Add the token + a webhook secret to .env

Generate a random secret string:

```bash
openssl rand -hex 32
```

Copy the output, then edit `.env` (in your repo root):

```
TELEGRAM_BOT_TOKEN=<paste-the-bot-token>
TELEGRAM_WEBHOOK_SECRET=<paste-the-output-of-openssl>
```

Restart paperclip so it picks up the new env vars:

```bash
docker compose up -d
```

## Step 3 — Start ngrok (the tunnel)

Install ngrok if you haven't yet:

```bash
brew install ngrok
```

One-time auth setup (paste your authtoken from
https://dashboard.ngrok.com/get-started/your-authtoken):

```bash
ngrok config add-authtoken <paste-here>
```

In a **new terminal tab** (keep this running while you test), point ngrok
at your paperclip on port 3001:

```bash
ngrok http 3001
```

You'll see output like:

```
Forwarding   https://abc123.ngrok-free.app -> http://localhost:3001
```

**Copy that `https://...ngrok-free.app` URL.** This is your public URL.

## Step 4 — Register the webhook with Telegram

Back in your main terminal (still in `~/blankcollar-agentic-os`):

```bash
./infra/scripts/setup-telegram.sh https://abc123.ngrok-free.app
```

(Replace with whatever ngrok gave you.)

You should see:

```
✓ Token valid — bot is @blankcollar_test_bot
✓ Webhook registered → https://abc123.ngrok-free.app/api/webhooks/telegram

Test it:
  1. Open Telegram, find @blankcollar_test_bot, send any message.
  2. Bot replies (via Hermes through Portkey).
  3. Watch `docker logs bc_paperclip --tail 30 -f` for inbound updates.
```

## Step 5 — Talk to your bot

1. Open Telegram → search for your bot's username (e.g. `@blankcollar_test_bot`).
2. Tap **START** (or send `/start`).
3. Type any message: `Hello, who are you?`
4. Within 5 seconds you'll get a reply from your bot — generated by
   Claude via Portkey, talking to you on behalf of your studio.

You can also watch it work in real time:

```bash
docker logs bc_paperclip --tail 30 -f
```

You'll see the inbound `POST /api/webhooks/telegram`, the Portkey call,
and the outbound `sendMessage` to Telegram.

## What's stored

Every message creates two real rows:

- `ops.goal` — title = first 200 chars of the message, kind=ephemeral,
  metadata.source=`telegram` + chat info.
- `ops.capture` — full text + parsed intent, links to the goal.

Plus an audit entry (`telegram.message`).

You can see them in the console at `localhost:3000`:

- **Activity** tab → your `telegram.message` entries.
- **Goals** tab → the goals created from your Telegram messages.

## Troubleshooting

- **`make doctor` shows paperclip unhealthy** — check
  `docker logs bc_paperclip --tail 50` for the boot error.
- **Bot doesn't reply at all** — likely the webhook isn't registered or
  ngrok stopped. Re-run setup-telegram.sh with a fresh ngrok URL.
  ngrok URLs change every time you stop + restart it (free tier);
  upgrade to a paid plan for stable subdomains, or use Cloudflare Tunnel.
- **Bot replies "Brain unavailable"** — Portkey isn't configured or
  your Anthropic key/credits ran out. Check your Portkey dashboard +
  Anthropic console.
- **Bot replies but it's slow (10s+)** — that's Portkey's first-call
  warmup. Subsequent calls are faster.
- **Telegram says "401 Unauthorized"** in setup — the bot token is wrong
  or has been revoked.

## Rotating the bot token

When you're done testing or if the token leaked:

1. Open Telegram, go to BotFather.
2. Send `/revoke`.
3. Pick your bot.
4. BotFather gives you a fresh token.
5. Update `TELEGRAM_BOT_TOKEN` in `.env`, restart paperclip, re-run
   `setup-telegram.sh`.
